# Stakeholder Agreement — Ikon Lot Scan (Lot Audit)

**Project:** Ikon Lot Scan (Lot Audit)  
**Date:** 2026-02-17  
**Purpose:** Align internal stakeholders on MVP scope. Only items explicitly stated as MVP are listed below.

---

## Initiative (what we’re building)

A standardized lot audit workflow for Ikon dealerships using rugged handheld VIN scanners. Field staff today move between multiple systems (PlanetX, Dealer Portal, Ikon Toolbox) to match vehicles to devices and check install status. VIN capture is inconsistent because label types and conditions (e.g. through glass, lighting) vary. Matching VIN to IMEI and device status is mostly manual, and audit outputs are not in a single standard format.

The initiative is to give field support a single, scanner-based flow: start an audit for a rooftop, scan VINs (barcode, QR, or plain text via OCR), get real-time VIN → IMEI lookup and a clear device status, with a defined action or resolution path for each status. Audit sessions capture who scanned, when, where, and how many vehicles, and the system produces standardized reports for field and corporate. Pilot in a small set of markets (Ikon Field Support only), with a path to broader rollout.

---

## IN SCOPE (MVP)

**Audit flow**

- Start scanning process; rooftop required before starting an audit.
- Scan VIN (barcode, QR, OCR).
- Real-time VIN → IMEI lookup.
- Device status classification with defined actions per status: Installed, Not Installed / Install Exception, Wrong Dealer, Not Reporting, Customer Linked / Registered, Invalid or Not Found.
- Record audit metadata: user, timestamp, geo, count (session counter).
- Generate standardized reports for FSMs and Corporate (e.g. CSV + summaries).

**Data displayed (per vehicle)**

- Serial  
- Activated date  
- Subscription expired  
- First reported  
- Last reported date  
- GPS Signal  
- Last known location  
- IP Port  
- Battery  
- Lat/long  
- Company  
- Group  
- Notes  

**Users & pilot**

- Ikon Field Support only.
- Two pilot markets (specific markets to be defined).
